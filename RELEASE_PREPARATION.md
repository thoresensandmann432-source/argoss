# 🚀 Подготовка проекта к финальному релизу

## 📋 Обзор

Этот набор инструментов поможет подготовить ваш Python проект к финальному релизу, автоматически исправив проблемы с кодировкой и проверив код на готовность.

## 🔧 Инструменты

### 1. `quick_fix.py` - Быстрое исправление
Исправляет конкретную проблему в файле `./src/interface/legacy/__init__.py`:
- Заменяет невалидный байт 0x97 на правильный символ em-dash (—)
- Создает файл с правильной UTF-8 кодировкой

**Использование:**
```bash
python3 quick_fix.py
```

### 2. `fix_encoding.py` - Исправление кодировки
Автоматически находит и исправляет все Python файлы с проблемами кодировки:
- Определяет текущую кодировку файла
- Конвертирует в UTF-8
- Заменяет проблемные символы Windows-1252 на правильные Unicode символы

**Использование:**
```bash
# Автоматически определяет директорию проекта
python3 fix_encoding.py

# Или указать директорию явно
python3 fix_encoding.py /path/to/project
```

**Что исправляется:**
- `\x97` → `—` (em dash)
- `\x96` → `–` (en dash)
- `\x93` → `"` (левая двойная кавычка)
- `\x94` → `"` (правая двойная кавычка)
- `\x91` → `'` (левая одинарная кавычка)
- `\x92` → `'` (правая одинарная кавычка)
- `\x85` → `…` (многоточие)

### 3. `validate_project.py` - Валидация проекта
Комплексная проверка проекта перед релизом:

**Проверки:**
- ✅ Синтаксис Python (AST + компиляция)
- ✅ UTF-8 кодировка всех файлов
- ✅ Доступность импортов
- ✅ Наличие requirements.txt
- ✅ Структура проекта
- ✅ Git статус (незакоммиченные изменения)

**Использование:**

### 4. `prepare_release.sh` - Автоматическая подготовка
Полностью автоматизированный процесс подготовки к релизу:

**Шаги:**
1. Быстрое исправление известных проблем
2. Исправление кодировки во всех файлах
3. Проверка синтаксиса Python
4. Финальная валидация
5. Проверка форматирования (если установлен Black)
6. Проверка типов (если установлен MyPy)

**Использование:**
```bash
chmod +x prepare_release.sh
./prepare_release.sh
```

## 🎯 Быстрый старт

### Вариант 1: Автоматический (рекомендуется)
```bash
# Скопируйте все скрипты в корень проекта
# Затем запустите:
chmod +x prepare_release.sh
./prepare_release.sh
```

### Вариант 2: Пошаговый
```bash
# 1. Исправьте конкретную проблему
python3 quick_fix.py

# 2. Исправьте все проблемы с кодировкой
python3 fix_encoding.py

# 3. Проверьте синтаксис
python3 -m py_compile src/**/*.py

# 4. Запустите валидацию
python3 validate_project.py
```

### Вариант 3: Только для GitHub Actions
Если ошибка возникает в GitHub Actions, добавьте в `.github/workflows/`:

```yaml
- name: Исправить кодировку
  run: |
    python3 quick_fix.py
    python3 fix_encoding.py

- name: Проверить синтаксис
  run: |
    find . -name "*.py" -type f ! -path "*/venv/*" ! -path "*/__pycache__/*" -exec python3 -m py_compile {} +
```

## 📝 Зависимости

### Обязательные:
- Python 3.7+
- Стандартная библиотека Python

### Опциональные (для fix_encoding.py):

### Опциональные (для дополнительных проверок):

## ⚙️ Интеграция с CI/CD

### GitHub Actions
```yaml
name: Release Preparation
on:
  push:
    branches: [main, release/*]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      
      - name: Install dependencies
        run: |
          pip install chardet
      
      - name: Fix encoding
        run: |
          python3 fix_encoding.py
      
      - name: Validate project
        run: |
          python3 validate_project.py
```

### GitLab CI
```yaml
validate:
  stage: test
  script:
    - pip install chardet
    - python3 fix_encoding.py
    - python3 validate_project.py
```

## 🐛 Устранение проблем

### Проблема: "unicode error 'utf-8' codec can't decode byte"
**Решение:**
```bash
python3 fix_encoding.py
```

### Проблема: "SyntaxError" после исправления
**Решение:**
```bash
# Проверьте конкретный файл
python3 -m py_compile path/to/file.py

# Запустите валидацию
python3 validate_project.py
```

### Проблема: Скрипты не находят проект
**Решение:**
```bash
# Явно укажите путь
python3 fix_encoding.py /full/path/to/project
python3 validate_project.py /full/path/to/project
```
